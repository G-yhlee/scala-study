# Email 설계할때 패턴매칭으로 할것인가 다형성으로 할것인가?

```md
visitor 트레이트와 , Anonymous , User 클래스가 있을때
User 에게 send email 기능을 추가하고 싶다고 할때

- visitor 트레이트에 다형성 패턴으로 기능을 추가하는것이 좋을지
- 아니면, EmailServie 객체를 만들어서 패턴매칭으로 기능을 추가하는것이 좋을지 생각해보자

// 결론
=> EmailService 객체를 만들어서 기능을 추가하는게 좋다
그이유는, visitor 트레이트와는 관련없는 email 관련된 데이터, 함수들이
추가될수있기 때문에 새로운 객체를 만들어 기능을 추가하는것이 좋다
```
